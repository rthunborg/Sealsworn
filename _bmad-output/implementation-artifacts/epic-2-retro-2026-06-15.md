## 3. What Went Well

Amelia (Developer): "Let's open with the wins, because there are real ones."

Cloud Dragonborn (Game Architect): "The headline: the domain-first boundary held under pressure. Epic 2 is the UI/accessibility/save epic — historically where teams let scenes start owning state. We did not. Every presentation surface is a `RefCounted` semantic contract — `TacticalBoardViewModel`, `TacticalLayoutProfile`, `TacticalAccessibilityModel` — proven scene-free in headless tests. Not a single `Control` node became tactical truth, and the command bridge is the only path from UI intent to mutation."

Charlie (Senior Dev): "And the contracts are pinned hard. `TacticalBoardViewModel.to_dictionary()` has an exact 16-key sorted assertion. When 2.5 added `layout` and 2.6 added `accessibility`, each addition had to intentionally bump the key count (15 → 16). That single test makes silent contract drift impossible — a key can't appear or vanish without a failing assertion. That's the kind of guardrail that pays off three epics from now."

Dana (QA Engineer): "Quality bar was genuinely high. Most stories cleared adversarial review on round one with zero patch items. The save/resume work especially — 2.7 and 2.8 came in with no-partial-corrupt-state assertions, real JSON round-trips, and first-divergence reporting baked in from the start, not bolted on after a reviewer complained."

Alice (Product Owner): "From a product-intent standpoint, two design promises got mechanically enforced, not just documented. The mobile two-step attack commit (2.3) means a mis-tap can never spend a turn or advance enemies — that's the anti-frustration pillar made real. And the difficulty non-goal (2.9) is enforced by a regression test asserting no `difficulty`/`enemy_scaling`/`damage_multiplier` key can ever enter settings. A future contributor literally cannot slip a difficulty slider in without a red test."

Amelia (Developer): "The standout breakthrough, though, was Story 2.7 surfacing the int64/JSON precision bug."

Charlie (Senior Dev): "Right — this is the gold from the dev notes. The RNG `root_seed` and per-stream `state` are 64-bit, but JSON numbers are IEEE-754 doubles with a 52-bit mantissa. `JSON.stringify`/`parse_string` was silently truncating any value past 2^53, and `RngStreamSet.try_restore` was rejecting JSON-parsed floats outright. That's a latent determinism-and-resume bug that survived all of Epic 1 — because Epic 1's Story 1.5 tests round-tripped *native dictionaries*, never real JSON. The fix was string-encoding those fields (read-tolerant of the old numeric form), and the durable lesson became a project-context rule: **always JSON-round-trip snapshots in tests, and string-encode any int64 save field.** We'd have shipped a save format that corrupted seeds the moment a run used a large seed."

Rasmus (Project Lead): "That's exactly the kind of thing the save foundation epic existed to catch before run structure depends on it. Good."

Cloud Dragonborn (Game Architect): "One more architectural win: settings landed as a *strictly isolated* subsystem. `user://settings.json` via `SettingsRepository`, with a cross-file test asserting a settings write never touches `user://run_autosave.json`. Two repositories, two files, zero contamination. That isolation is what lets Epic 8's profile/meta save be a third independent file later without entanglement."

## 5. Key Insights / Lessons Learned

1. **Test against the real serialization boundary, not a convenient proxy.** The int64 bug hid for an entire epic behind native-dict round-trips. Any save/snapshot test must `JSON.stringify` → `parse_string` and assert the restored RNG reproduces the *exact next draw*. (Now codified in `project-context.md`.)
2. **Pin presentation contracts with exact-shape assertions.** The 16-key `to_dictionary()` assertion turned every view-model slot addition into a deliberate, reviewed act. This pattern should extend to every new semantic contract.
3. **Enforce design non-goals as executable tests, not prose.** The difficulty-ladder forbidden-keys test is the model: when a design boundary matters (here FR90/FR91/FR93), a red test is worth more than a paragraph.
4. **Domain-first survives the UI epic if you build semantic contracts before scenes.** Deferring `Control` presenters kept tactical truth out of the scene tree and kept the whole epic headless-testable. Worth repeating for every future UI surface.
5. **Harness/tooling failures masquerade as model failures.** The 2.5 halt cost cycles to the wrong diagnosis. When a delegate "silently" misbehaves, suspect frontmatter/schema validation before model/effort tuning.
6. **Repeated review feedback is a refactor signal.** When the same hardening note (value sanitization) recurs across stories, that's the codebase asking for a consolidation, not N more local patches.

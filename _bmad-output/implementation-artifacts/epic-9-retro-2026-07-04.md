## 1. Epic Summary

Epic 9 is the epic that gave the MVP run an **endpoint worth reaching** — a real, live, tactical boss fight against the Larval Avatar, a first-victory reveal, and a return to the roguelite loop. It landed in five deliberate layers, each building strictly on the seam the prior one exposed: **9.1** the boss-encounter SETUP (a dedicated `BossNodeEnterCommand` that, on boss-node arrival, builds a `BossEncounterRequest` + a deterministic `BossArenaBuilder` arena carrying a boss-entity SLOT MARKER, transitions `ACTIVE_ROUTE → NODE_RESOLUTION`, emits the append-only `boss_encounter_started` event, and parks the run non-terminal — replacing the v0 auto-completing boss placeholder without touching the route model, the boss node type, or the `run_completed`/`boss_placeholder` boundary); **9.2** the boss CONTENT + PHASE MODEL (a validated `BossDefinition` + `BossPhaseDefinition` + `BossActionDefinition` code-constant family on a fail-loud `BossRepository`, filling the 9.1 slot at `larval_avatar`, plus a pure, deterministic, forward-only, idempotent `BossPhaseResolver` and the append-only `boss_phase_changed` event); **9.3** the live FIGHT behavior (the Larval Avatar becomes a live board entity — `BossAi` phase-constrained ZERO-RNG utility scorer, `BossCommandAdapter` routing actions through existing events, `BossTurnResolver` simulate-then-apply turn driver + the live phase re-resolution seam, and a two-turn telegraph→resolve flow reusing the Ash Seer precedent); **9.4** the VICTORY resolution (the `detect_boss_defeat` seam + a `boss_defeated` event + the `victory` `run_completed` outcome + `resolve_run_end(victory)` + the eligibility-independent `first_victory_recorded` latch/command/event + the `FirstVictoryRevealBeat` DTO, mirroring 8.5's ratified first-DEATH precedent at the OPPOSITE terminal phase); and **9.5** the finale closeout (an approved 5-seed boss-seed regression catalog through the whole chain, a full-run-through-the-shell integration proving `boss_cleared` + the correct completion event, and a local/offline build-profile-gated `BossAttemptDiagnostics` tuning recorder).

The epic held to its plan with the same structural fidelity that carried Epics 5–8 — but this time while crossing the single hardest seam the whole project had parked: the **live tactical-play loop**. Epic 8's retrospective (T1) called this the headline latent debt and predicted it would come due in Epic 9. It did, and it was crossed cleanly:

## 5. Key Insights

1. **Stage a live system behind headless layers whose OUT-of-scope boundaries end where the next layer's dependency begins.** The boss reached the live board only in 9.3, but its arena (9.1), definition (9.2), phase resolver (9.2), and events (9.1/9.2) were all authored and validated headless first — each story's boundary drawn exactly at the next's dependency. That is why the hardest-parked seam in the project (the live tactical loop) landed additively instead of as a rewrite. The general lesson: when a big live system is coming, ship its pure/content/state-machine substrate in separate headless stories FIRST, so the live-loop story is a thin instantiation, not a big-bang.
2. **When you replace a v0 placeholder with a live path, re-audit every DERIVED read that keyed off the placeholder's SIDE EFFECTS.** The placeholder boss command cleared the boss route node as a side effect of completing the run; the live victory command (correctly) does not, because clearing a boss node is a boss-specific step the generic completion command must not know about. `RunSummary.boss_cleared` derived from that clear, so it silently broke until 9.5 moved the clear to a caller-driven continuation. The trap is invisible unless you enumerate what the placeholder's side effects were and check each downstream reader.
3. **Flag a cross-story seam risk at the story where it is BENIGN, and turn it into a caller contract, so the story where it turns DANGEROUS just consumes the contract.** The sequence-id collision was harmless in 9.3 (system events, un-interleaved) and dangerous in 9.4 (interleaved streams). Rather than defer it, 9.3's review made `resolve_phase_transitions` take a caller-reserved base + return a cursor + document the seam contract; 9.4 threaded the shared counter; 9.5 re-affirmed with a no-collision test. Catching it where it's benign is cheaper than debugging a duplicate id in an interleaved log later.
4. **Recompute derived state from its source instead of persisting it, and the save gate never moves.** The boss's current phase is a pure function of its current HP (`active_phase_index(definition, current_hp)`), recomputed at each boss turn — so no phase field was persisted, the 23-key `RunSnapshot` gate stayed 23, and `boss_phase_changed` events remain the honest crossing record. Persisting the phase would have been a save-key addition and a resume-consistency burden for zero benefit.
5. **An additive schema field at v1 beats a schema bump — but only if you reconcile it against the existing migration matrix.** `first_victory_recorded` had no pre-reserved home (unlike 8.5's first-death), so it was added as a NEW additive field at `SCHEMA_VERSION == 1`, lenient-parsed (a pre-9.4 dict → false), with the `DICTIONARY_KEYS` pin updated but NO bump. The load-bearing check was reconciling it with 8.7's already-green migration tests (which pin `SCHEMA_VERSION == 1` and reject `schema_version: 2`) — a lenient additive field at v1 keeps both pins green; a bump would have silently broken them.
6. **The finale's "determinism" is REPRODUCIBILITY, not divergence-catching — and that's the correct design for a ZERO-RNG boss.** The 5-seed regression catalog runs each path twice and compares byte-identity, but because the arena is a fixed seed-independent layout and the boss AI is ZERO-RNG, all seeds behave near-identically (only the `arena_seed` decimal string differs). There is no layout fingerprint to dump; the "fingerprint" is a composite of the deterministic setup/phase/telegraph/outcome records plus a preserved annotated tuning catalog. Naming that honestly (a reproducibility pin + a tuning artifact, not a divergence detector) is more useful than pretending the catalog catches drift it structurally cannot.
7. **"Reachable via the shell + provable in an integration test" is a distinct, legitimate milestone from "playable hands-off."** Epic 9 drives the full run to the boss through `run_to_completion`, fights the boss on the live board via explicit turns, and resolves victory/death via the 9.4 seam — a real end-to-end integration. It deliberately does NOT auto-play the fight inside `run_to_completion` or wire a live hero-death source. Distinguishing these two milestones (and getting a human to ratify the split) is what let the epic land the finale's LOGIC completely while honestly deferring its live PLAYABILITY to the run-flow/HUD story — without either over-building or leaving a silent gap.

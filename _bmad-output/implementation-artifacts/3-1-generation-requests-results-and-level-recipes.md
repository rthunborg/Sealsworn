# Story 3.1: Generation Requests, Results, and Level Recipes

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a player,
I want generated levels to come from explicit recipes and seeds,
so that each level can be reproduced, tested, and debugged.

## Acceptance Criteria

1. Given a generation request is created, when it is initialized, then it includes root seed or level seed, node context, level size, difficulty band, affinity placeholder, and constraints, and it uses the named `level` RNG stream for layout-affecting randomness.
2. Given a level recipe is loaded, when generation begins, then the recipe defines size class, terrain rules, blocker budget, enemy budget, reward placement rules, and tactical wrinkle requirements, and gameplay systems access it through repository or fixture boundaries.
3. Given a generator needs enemy, reward, affinity, or recipe definitions, when generation phases run, then definitions are selected through approved repository/import boundaries, and the generator does not read arbitrary JSON or CSV files directly in the hot path.
4. Given generation succeeds or fails, when a result is returned, then it uses a structured `GenerationResult`, and failures include phase, seed, reason, and compact diagnostics.

### Generation Contract (AC1 + AC4 — request sources `level` RNG; result is structured success-or-error)

The single most important determinism constraint: **layout-affecting randomness comes from the named `level` RNG stream, never global `randi()`/`randf()` and never another stream — and every generation outcome is a structured `GenerationResult` carrying enough to reproduce and debug it.**

- `GenerationRequest` carries the seed + level context (root/level seed, node context, recipe id, size class, difficulty band, affinity placeholder, constraints). The recipe is referenced BY ID and resolved through `LevelRecipeRepository` — the request does NOT embed raw recipe data or read a file (AC3). [Source: epics.md Story 3.1 AC1; game-architecture.md#Generation Pipeline Pattern "GenerationRequest: seed, route node, level recipe, difficulty, constraints"]
- Layout RNG: any draw that affects the generated layout MUST go through `RngStreamSet.rand_int(RngStreamSet.STREAM_LEVEL, ...)` / `rand_float(RngStreamSet.STREAM_LEVEL, ...)`. Prove with a test: same root seed -> same `level`-stream state -> identical draw sequence; and that unrelated `cosmetic`/`combat` draws do NOT change the `level`-affected outcome (the Story 1.4 stream-isolation contract carried into generation).

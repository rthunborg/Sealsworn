# Story 7.4: Affinity Definitions and Assignment

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a player,
I want levels to carry readable affinity identities,
so that Scorched, Flooded/Conductive, Cursed, and Darkness change tactical expectations.

## Acceptance Criteria

(Verbatim from `planning-artifacts/epics.md` lines 1942-1957, Story 7.4; FR56 — "MVP affinities must support Scorched, Flooded/Conductive, Cursed, and Darkness". The FIRST of the three affinity stories: 7.4 = definitions + deterministic assignment + the neutral/no-affinity contract; 7.5 = tactical effects; 7.6 = Darkness fairness. Builds on the Epic-3 generation pipeline + the Epic-4 run/route layer + the 7.1-7.3 risk-economy domain.)

**AC1 — MVP affinity definitions expose id, display name, tactical rules, visual tags, and explanation text; invalid affinity content fails validation**
- **Given** MVP affinity definitions are authored
- **When** the content repository loads them
- **Then** Scorched, Flooded/Conductive, Cursed, and Darkness definitions expose id, display name, tactical rules, visual tags, and explanation text
- **And** invalid affinity content fails validation.

**AC2 — A generated/selected level receives a deterministic affinity recorded in the level snapshot**
- **Given** a generated or selected level receives an affinity
- **When** assignment runs
- **Then** the affinity is recorded in the level snapshot
- **And** assignment is deterministic for the same seed and route state.

**AC3 — A level with no affinity yields an empty/neutral rule set with no side effects**
- **Given** a level has no affinity
- **When** tactical systems query affinity rules
- **Then** they receive an empty or neutral rule set
- **And** no affinity side effects occur.

## Scope boundary (read FIRST — this story is AFFINITY CONTENT + DETERMINISTIC ASSIGNMENT + the NEUTRAL contract: the 4 MVP affinity definitions, a fail-closed repository, a deterministic per-level assignment that RECORDS the affinity id in the level snapshot, and the empty/neutral rule-set contract. It does NOT make affinities ALTER tactical play — that is 7.5/7.6)

7.1-7.3 built the risk-economy DOMAIN (`RiskEconomyState`, curse/corruption rules, the `events`-stream event-choice flow). 7.4 is the FIRST of the three AFFINITY stories. The generation/run layers ALREADY carry the affinity SEAMS — 7.4 fills them with real content + a real assignment:

**OUT of scope (explicitly later stories / NOT this story — do NOT pull forward):**
- **Affinities ALTERING tactical play** (Scorched DoT/hazard damage, Flooded/Conductive pathing pressure/electric interactions, Cursed risk/reward hooks firing through the rules kernel) → **Story 7.5** (FR57, "Affinities must alter tactical choices rather than only visuals"). 7.4 authors the DEFINITIONS + the deterministic ASSIGNMENT + the RECORD; it does NOT make an active affinity change a combat HP/damage number, fire a hazard, or apply a kernel hook. 7.4's `tactical_rules` is DATA (a recorded description of intent) the 7.5 effects layer will READ — 7.4 is the PRODUCER of the assigned affinity + its declared rules, NOT the consumer that applies them. `scripts/rules/{conditions,operations}` STAY EMPTY (0 `.gd` each — the per-effect operation engine that makes an affinity rule change a live number is the later operations/7.5 story).

- **The HUD `.tscn` for affinity display / the affinity badge on the run map / inspect-panel affinity surface** → a later HUD story (UI-scene-last; the SAME residual every Epic 5/6/7 story left). 7.4 builds the definition + the assignment + the RECORD + OPTIONALLY a scene-free `AffinityViewModel` read surface (mirroring `CursedRewardViewModel`, IF an AC1 readable surface is wanted as data); a later HUD story owns the scene + the badge. NONE of the three ACs demands an affinity fire a tactical effect (that is 7.5/7.6, FR57/FR58). The "tactical rules" in AC1 are the RECORDED rule data the 7.5 layer will consume — 7.4 produces and validates that data, it does not execute it.

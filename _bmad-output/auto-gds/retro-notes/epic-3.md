## Story 3-2-seed-stable-small-level-layouts
- [Phase 5 — dev-story] Small (8x8) reachability is guaranteed STRUCTURALLY in v0, not validated: full border-wall ring + open interior + a blocker-free reserved central corridor row between a FIXED entrance (1, h/2) and exit (w-2, h/2). The 3.1 `{placeholder: true}` payload is gone — later stories reading the payload must use `payload.board` / `payload.layout_pending`, not `payload.placeholder`.

- [Phase 3 — create-story] Story 3.3 is the FIRST epic-3 story that must REJECT layouts with diagnostics (excessive blockage / unreachable exit / unreadable first reveal) — 3.2 only guaranteed fairness structurally. Scope this tightly to AC2's three named checks; the general validator+bounded-retry framework is still Story 3.6. The AC2 excessive-blockage bound is what guards readability.
- [Phase 5 — dev-story] LevelGenerator.generate() Medium payload now returns a REAL layout `{board, entrance, exit, blockers, size_class, recipe_id, level_seed}` (GenerationResult.ok diagnostics phase "validation"), replacing 3.2's `{layout_pending: true, ...}`.

## Story 4-1-run-state-and-route-node-model
- [Phase 3 ‚Äî create-story] Story directs a NEW `scripts/run/` domain folder for the run-progression model (`RunState`/`RouteState`/`RouteNode`) and adding `run` to the project-context.md code-organization domain list. Architecture specifies the `RunState` machine but maps no directory for the run model (route *generation* ‚Üí `scripts/generation/route/`, saves ‚Üí `scripts/save/`). Defensible (mirrors how `settings` was added in Epic 2) but is a genuine convention extension later stories inherit ‚Äî flag if a different placement is preferred.
- [Phase 5 ‚Äî dev-story] Run phase is serialized as `run_phase` NESTED inside the existing `route_state` payload, deliberately NOT a new top-level `RunSnapshot` key ‚Äî keeps the pinned 23-key no-surprise gate green untouched. Later route stories must keep run-progression fields under `route_state`, not flatten new top-level run keys onto `RunSnapshot`.
- [Phase 5 ‚Äî dev-story] `run_started` `DomainEvent` is now fully wired (factory + payload validator, system event, no actor) and tested, but has NO emission site yet ‚Äî no run-start command exists in 4.1. A later Epic 4 story (run-start command) must actually emit it, or the validator is dead weight.
- [Phase 7 ‚Äî code review] `DomainEvent._has_decimal_string_payload` (the new `run_started` `root_seed` validator) uses loose `is_valid_int()` only ‚Äî it accepts out-of-int64 strings that `to_int()` saturates/wraps, weaker than the strict lossless-decimal-string check Story 3.7 added to `ManualSeedLoader.parse_seed`. It is the project's first seed-string validator added AFTER 3.7 set the lossless idiom and it diverged. Deferred to the first `run_started` emitter (folds with the `node_count` defer). Epic-retro lesson: any new seed-string validator must adopt the 3.7 lossless/canonicalize idiom, not bare `is_valid_int()`.

## Story 4-2-seeded-8-12-node-route-generation
- [Phase 3 ‚Äî create-story] 4.2 structurally bounds the route to 8-12 non-boss nodes (`run_started.node_count` ‚àà [8,12]), so the eventual run-start emitter (4.3/4.6) can close the two `run_started` int64/lossless-decimal defers as permanently benign rather than re-litigating them.
- [Phase 5 ‚Äî dev-story] `RouteGenerator.INTERIOR_COLUMN_COUNT = 6` (not 7) is load-bearing for the "‚â•1 branch point per seed" guarantee (AC4): min interior demand (7 nodes) > 6 columns forces ‚â•1 width-2 column even at the count=8 floor; 7 columns collapses to an all-linear route. Any column-count/band change must re-verify the branch-point guarantee and re-pin route fingerprints via `tools/dump_route_fingerprints.gd`.
- [Phase 5 ‚Äî dev-story] Route result-shape pattern ratified: `RouteGenerator` returns a `GenerationResult` (`PHASE_ROUTE`) whose serializable `payload.route_state` rehydrates via `route_from_result` ‚Äî payload carries the snapshot dict, not the live `RefCounted`. The run-start command (4.3/4.6) must consume it by rebuilding the `RouteState`, mirroring the level pipeline discipline.
